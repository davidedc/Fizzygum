# Phase S — static catalog of layout-enqueue origins

**What this is.** The complete static enumeration of every code path that can put a widget on
`world.widgetsThatMaybeChangedLayout` (the layout-invalidation queue), with an a-priori classification. This is the
"universe of possible origins"; the runtime audit (Phase R, see `end-of-cycle-flush-inventory.md`) measures which
actually **survive** to the end-of-cycle flush, and Phase X flags any catalog entry the suite never exercised.

Generated by grepping `Fizzygum/src` for `invalidateLayout` callers + the re-fit seam feeders
(`_reFitContainer` / `_reFitContainerAfterRawGeometryChange` / `_refreshScrollPanelWdgtOrVerticalStackIfIamInIt`).
**84 call sites** total. Lines drift — grep the method name.

## How a widget enters the queue (recap)

`Widget.invalidateLayout` (`src/basic-widgets/Widget.coffee` ~:3749) pushes the widget if its layout was valid,
then **climbs** to the parent by self-recursion. The re-fit **seam** `_reFitContainer` (~:1642) is the other entry:
its **out-of-pass** arm calls `container.invalidateLayout()`; its **in-pass** arm pushes directly (legal mid-pass).
A push survives to the end-of-cycle flush **iff** no self-settling flush (`mutateGeometryThenSettle` /
`settleLayoutsOnceAfter`) drains the queue between the push and `WorldWdgt.doOneCycle`'s `recalculateLayouts()`.

## A-priori buckets

- **(A) Inside a public self-settling setter** — wrapped in `mutateGeometryThenSettle`, so it self-drains before
  returning. **Not a category-2 source; excluded** (will not survive to end-of-cycle in steady state).
- **(B) Raw/silent/fullRaw mutator or seam plumbing** — reaches end-of-cycle only when run outside a flush.
  Continuous gestures (drag) and low-level plumbing. The *callers* matter.
- **(C) Direct discrete feature-code caller** — a menu pick, button, toggle, collapse, property edit, re-parent,
  drag/drop event. **The prime "should it self-settle?" candidates.**
- **(D) Construction / build / factory** — invalidates during `buildAndConnectChildren`/`constructor`/`createToolsPanel`
  while the widget is still an **orphan** (not yet in the world). The orphan guard in `mutateGeometryThenSettle`
  skips the flush; these settle when the finished widget is added. Expected at **boot**, not steady-state interaction.

**Headline of the a-priori pass:** the vast majority of the 84 sites are **(A) self-settling** or **(D) construction**.
The genuine **(C) discrete** suspects number ~25 methods, of which the **VerticalStackLayoutSpec property setters**
are the textbook convert-to-self-settle candidates. The runtime audit (Phase R) decides which of these actually
reach end-of-cycle on *interaction* frames.
